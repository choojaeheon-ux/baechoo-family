# 배추 예방접종 체크리스트 설계

- 작성일: 2026-06-30
- 대상: 배추가족 PWA — 배추 생활기록부 / 건강 탭
- 목적: 강아지 배추의 예방접종(매년 추가접종)을 표준 백신 프리셋 기반 체크리스트로 관리

## 배경

건강 탭(`HealthList`)은 현재 두 영역으로 구성:
1. **할 일**(`HealthTodos`) — once(약·접종 예정일 D-day, 1회 완료) / daily(양치 등) 범용 체크
2. **건강 기록** — healthType 8종 기록 리스트 + 리스트/캘린더 토글

기존 할 일의 once는 "한 번 완료하면 끝"이라 매년 반복되는 예방접종 부스터에 맞지 않는다. 예방접종 전용 섹션을 신설해 **반복 주기·접종 이력**을 관리한다.

## 결정 사항 (확정)

| 항목 | 결정 |
| --- | --- |
| 항목 구성 | 표준 백신 프리셋 (성견 연례 부스터 5종) |
| 반복 주기 | 매년 부스터 자동 (체크 시 다음 예정일 +1년) |
| 위치 | 건강 탭 내 신설 섹션 (서브탭 추가 아님). "할 일" 아래 / "건강 기록" 버튼 위 |
| 건강 기록 연동 | 안 함 (체크리스트 자기완결, 중복 방지) |
| 프리셋 제공 | "+ 표준 백신 추가" 버튼으로 5종 일괄 시드. 빈 상태에서 시작 |

## 표준 백신 프리셋 (5종)

성견 연례 추가접종 기준. interval = 12개월.

1. 종합백신 (DHPPL)
2. 코로나 장염
3. 켄넬코프 (기관지염)
4. 인플루엔자
5. 광견병

## 데이터 모델

신규 엔티티 `BaechooVaccine` (우주 체크리스트와 동일한 표준 경로).

```ts
export interface BaechooVaccine {
  id: string;
  name: string;           // 백신 이름
  lastDone: string | null;   // 마지막 접종일 (YYYY-MM-DD, 미접종이면 null)
  nextDue: string | null;    // 다음 예정일 (YYYY-MM-DD). 미접종이면 null
  intervalMonths: number;    // 추가접종 주기 (기본 12)
  history: string[];         // 접종일 누적 (YYYY-MM-DD 오름차순)
  memo: string | null;
  createdAt: string;         // YYYY-MM-DD
}
```

- `DataSnapshot`에 `baechooVaccines: BaechooVaccine[]` 추가.

## UI 동작

### 섹션 컴포넌트 `VaccineList` (`components/baechoo/VaccineList.tsx`)

`HealthList`에서 `<HealthTodos />` 아래, "+ 건강 기록" 버튼 위에 `<VaccineList />` 렌더.

- 섹션 제목 "예방접종" + 우측 "+ 백신" 버튼(직접 추가 폼).
- 백신이 0개일 때: 안내 카드 + **"표준 백신 추가"** 버튼 → 프리셋 5종 일괄 생성(미접종 상태). 이미 존재하는 이름은 건너뜀(중복 방지).
- 백신 ≥1개일 때: 다음 예정일 가까운 순(미접종은 뒤) 카드 리스트.

### 백신 카드

```
[✓]  종합백신 (DHPPL)              [D-12]   ← Pill
     마지막 접종 2025-07-02
```

- 체크 동그라미(✓, `HealthTodos`의 `Check` 패턴 재사용).
- 이름 탭 → 편집/삭제 폼.
- D-day Pill: nextDue 기준 `dday`/`ddayLabel`. 톤 = 지남 coral / ≤30일 gold / 그외 stone. 미접종이면 "미접종" stone Pill.
- 마지막 접종일 표시(없으면 "접종 기록 없음").

### 체크(접종 완료) 동작

체크 ✓ 누르면:
1. `lastDone = 오늘`
2. `history`에 오늘 추가
3. `nextDue = 오늘 + intervalMonths개월`

→ D-day가 다음 해로 갱신되며 카드가 "완료/리셋"되는 게 아니라 다음 주기로 넘어간다. (UI상 잠깐 완료 표시 후 다음 예정으로 갱신)

### 편집 폼 `VaccineForm` (`forms.tsx`)

- 필드: 이름, 마지막 접종일(date, 선택), 다음 예정일(date, 선택), 주기(개월, 기본 12), 메모.
- 삭제는 2단계 확인(기존 `DeleteButton` 패턴).

## 비범위 (YAGNI)

- 건강 기록 자동 생성(연동) — 하지 않음.
- 접종 알림/푸시 — 하지 않음.
- 백신별 상세 이력 화면 — history 배열만 저장, 전용 화면은 추후.
- 강아지 나이 기반 1차 접종 스케줄(퍼피 시리즈) — 성견 부스터에 집중, 추후.
