# 지역화폐 거래내역 연동 + 대시보드 이동 — 설계

- 작성일: 2026-06-18
- 대상 앱: 배추가족 PWA (`baechoo-family`)
- 범위: 가계부 지역화폐 기능 2건

## 배경

지역화폐(온누리·경기지역화폐 등)는 선불 지갑(잔액 관리)으로 현재 **고정지출 탭** 안에 있다.
두 가지 불편이 있다.

1. 지역화폐를 충전/사용해도 가계부 거래내역과 연동되지 않아 실제 지출에 반영되지 않는다.
2. 고정지출 탭 하단에 묻혀 있어 잔액 현황을 한눈에 보기 어렵다.

## 결정 사항 (사용자 확정)

| 항목 | 결정 |
|------|------|
| 지출 시점 | **충전할 때** 지출로 기록 (충전 = 현금 유출). 사용은 잔액 차감만, 거래 미기록 |
| 충전 거래의 카테고리·결제수단 | **지역화폐별 기본값** — 지역화폐 추가/수정 시 지정, 충전 시 자동 적용 |
| 위치 이동 | 지역화폐 섹션을 **대시보드로 통째 이동** (충전/사용/추가/수정 전체), 고정지출 탭에서는 제거 |
| 잔액 정합성 | 충전 거래에 지역화폐 링크를 박아, 거래내역에서 **삭제하면 잔액 자동 보정**(충전 취소). 금액 수정 보정은 범위 외 |

회계 모델: 충전 = 현금 → 지역화폐 자산 이동이지만 가계부상 그 시점에 돈이 나간 것으로 본다.
사용은 이미 충전 때 잡힌 지출이므로 **이중 계상 방지**를 위해 거래를 만들지 않는다.

## 기능 1 — 충전 시 거래내역 자동 연동

### 데이터 모델 변경

`LocalCurrency`에 충전 거래 기본값 2개 추가:

```ts
export interface LocalCurrency {
  id: string;
  name: string;
  balance: number;
  monthlyCharge: number;
  defaultCategoryId: string | null;       // 충전 거래에 쓸 기본 카테고리
  defaultPaymentMethodId: string | null;  // 충전 거래에 쓸 기본 결제수단
}
```

`Transaction`에 지역화폐 역링크 1개 추가(삭제 시 잔액 보정용):

```ts
export interface Transaction {
  // ...기존 필드
  localCurrencyId: string | null; // 이 거래가 어떤 지역화폐 충전으로 생성됐는지(아니면 null)
}
```

### DB 마이그레이션 (`0005_local_currency_link.sql`)

```sql
alter table local_currencies
  add column if not exists default_category_id text,
  add column if not exists default_payment_method_id text;

alter table transactions
  add column if not exists local_currency_id text;
```

- 기존 운영 규칙대로 **사용자가 Supabase SQL Editor에 직접 실행**한다(연결된 MCP는 회사 계정).
- localStorage 모드는 객체를 그대로 저장하므로 별도 작업 불필요.

### 영속화 매핑 (`lib/repo.ts`)

`toLc`/`fromLc`에 두 컬럼, `toTxn`/`fromTxn`에 `local_currency_id` 매핑 추가:

```ts
// toLc / fromLc
defaultCategoryId: (r.default_category_id as string) ?? null,
defaultPaymentMethodId: (r.default_payment_method_id as string) ?? null,
default_category_id: x.defaultCategoryId,
default_payment_method_id: x.defaultPaymentMethodId,
// toTxn / fromTxn
localCurrencyId: (r.local_currency_id as string) ?? null,
local_currency_id: x.localCurrencyId,
```

### 폼 변경 (`LocalCurrencyForm`)

- 카테고리 select 추가 (지출 카테고리 목록, 기본 선택 = 첫 지출 카테고리)
- 결제수단 select 추가 ("선택 안함" 포함)
- 저장 시 `defaultCategoryId`, `defaultPaymentMethodId` 함께 저장

### 충전 동작 변경

`+충전` 시트(`AmountSheet`) 확정 시 기존 잔액 증가에 더해 지출 거래를 자동 생성한다.

```
잔액: balance += amt  (기존 유지)
거래 자동 생성(saveTransaction):
  type: "expense"
  amount: 충전액
  date: 오늘(todayISO)
  categoryId: lc.defaultCategoryId ?? 첫 지출 카테고리(fallback)
  paymentMethodId: lc.defaultPaymentMethodId   (null 허용)
  memo: `${lc.name} 충전`
  member: "chuchu"
  source: "auto"
  isSpecial: false
  habitTag: null
  recurringId: null
  localCurrencyId: lc.id   // 삭제 시 잔액 보정을 위한 역링크
  isPaid: true
```

- `defaultCategoryId`가 비어 있는 기존 지역화폐도 안전하게 동작하도록 fallback(첫 지출 카테고리) 적용.

### 삭제 시 잔액 보정 (`lib/data-context.tsx`)

`removeTransaction(id)`이 현재 id만 받으므로, 삭제 전 state에서 해당 거래를 조회해 보정한다.

```
removeTransaction(id):
  const tx = transactions.find(t => t.id === id)
  await repo.deleteTransaction(id)
  setTransactions(filter out id)
  if (tx?.localCurrencyId) {
    const lc = localCurrencies.find(l => l.id === tx.localCurrencyId)
    if (lc) await saveLocalCurrency({ ...lc, balance: Math.max(lc.balance - tx.amount, 0) })
  }
```

- 충전 거래를 지우면 충전했던 금액만큼 잔액이 줄어든다(충전 취소). 잔액은 0 미만으로 내려가지 않는다.

### 사용 동작 (변경 없음)

`−사용`은 기존대로 `balance -= amt`(0 미만 방지)만 수행, 거래 미생성.

### 범위 한계 (v1, 명시)

- **삭제**는 잔액을 자동 보정하지만, 거래내역에서 충전 거래의 **금액을 직접 수정**하는 경우는 잔액 보정 대상이 아니다(드문 케이스). 충전액을 바꾸려면 해당 거래를 삭제하고 다시 충전한다.
- 지역화폐 자체를 삭제하면 그 지역화폐로 만든 과거 충전 거래는 기록으로 남는다(링크만 끊김).

## 기능 2 — 지역화폐 섹션 대시보드 이동

### 이동 대상

`FixedExpenses.tsx`의 지역화폐 관련 전부를 `Dashboard.tsx`로 옮긴다.

- UI: 섹션 타이틀 + `+추가` 버튼, 지역화폐 카드 목록(잔액·매월 충전 표시, +충전/−사용 버튼)
- state: `lcOpen`, `editLc`, `chargeLc`, `useLc`
- 컴포넌트: `LocalCurrencyForm`, 충전/사용 `AmountSheet` 2개
- `FixedExpenses.tsx`에서는 해당 섹션·state·시트를 **제거**(다른 섹션은 그대로)

### Dashboard 데이터 의존성 추가

`useData()`에서 추가로 사용: `localCurrencies`, `saveLocalCurrency`, `saveTransaction`, `categories`, `paymentMethods`(폼 select용).
충전 거래 생성 로직(fallback 카테고리 계산 포함)은 Dashboard 내부에 둔다.

### 배치 위치

대시보드 상단 "이번 달 예산" 요약 **다음**에 지역화폐 섹션을 배치한다(잔액 = 즉시 쓸 수 있는 돈 성격이라 상단 노출). 최종 위치는 사용자 검토 시 조정 가능.

- `tsc --noEmit` + `next build` 통과
- 수동 확인:
  - 지역화폐 추가(카테고리/결제수단 지정) → +충전 → 거래내역에 `OO 충전` 지출 생성 + 대시보드 예산 사용액 반영

  - −사용은 잔액만 차감(거래 없음)
  - 대시보드에서 충전/사용/추가/수정 동작
