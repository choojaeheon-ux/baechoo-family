# 매달 반복 예산 (기본 예산 + 월 오버라이드) — 설계 명세

- 작성일: 2026-07-03
- 상태: 설계 확정 (구현 착수 전)
- 대상: 배추가족 PWA 가계부 — 예산·목표 탭(`components/budget/Plans.tsx`)

## 1. 목적

현재 예산은 `yearMonth`로 묶여 있어 **매달 새로 설정**해야 한다. 매달 거의 같은 예산을 반복 입력하는 번거로움을 없앤다.

**목표**: 예산을 한 번 설정하면 매달 자동 적용되고, 특정 달만 다르게 쓸 일이 있을 때 그 달·그 카테고리만 수정한다.

## 2. 핵심 개념

- **기본 예산(default)**: 매달 자동 적용되는 카테고리별 예산. 한 번 설정.
- **월 오버라이드(override)**: 특정 달·특정 카테고리만 기본과 다른 값. 그 달에만 적용.
- 오버라이드는 **카테고리 단위**로 독립 — 6월 식비만 40만으로 바꿔도, 그 달 나머지 카테고리는 기본값을 그대로 쓴다.

## 3. 데이터 모델

기존 `budgets` 테이블을 재사용한다. `Budget.yearMonth`를 **nullable**로 확장한다.

```ts
export interface Budget {
  id: string;
  yearMonth: string | null; // null = 기본 예산(매달), "YYYY-MM" = 그 달 오버라이드
  categoryId: string | null; // null = 전체 월예산, 그 외 = 카테고리별
  amount: number;
}
```

- `yearMonth = null` → 기본 예산
- `yearMonth = "2026-06"` → 2026-06 오버라이드
- `(categoryId, yearMonth)` 조합이 한 예산 레코드를 유일하게 식별한다(같은 카테고리에 기본 1개 + 달마다 오버라이드 0~1개).

**마이그레이션**: `budgets.year_month`를 `not null` → `nullable`로 (`alter table budgets alter column year_month drop not null`). 기존 데이터는 전부 `"YYYY-MM"`이라 그대로 보존되며, null(기본)은 신규 입력분부터 생긴다.

52주 투두의 `week_num` nullable(날짜 미정)과 동일한 선례.

## 4. 조회 로직 (`lib/compute.ts`)

"그 달 오버라이드가 있으면 그 값, 없으면 기본값" 규칙을 두 헬퍼에 적용한다.

```ts
// 카테고리 예산: 오버라이드 우선, 없으면 기본
export function budgetForCategory(
  budgets: Budget[], ym: string, categoryId: string | null,
): number {
  const override = budgets.find(
    (b) => b.yearMonth === ym && b.categoryId === categoryId,
  );
  if (override) return override.amount;
  const base = budgets.find(
    (b) => b.yearMonth === null && b.categoryId === categoryId,
  );
  return base?.amount ?? 0;
}

// 전체 월예산: 전체(categoryId=null) 오버라이드/기본 우선, 없으면 카테고리별 합
export function totalBudget(budgets: Budget[], ym: string): number {
  const overall = budgetForCategory(budgets, ym, null); // null 카테고리 = 전체
  if (overall > 0) return overall;
  // 카테고리별 합: 그 달 실제 적용값(오버라이드 우선)으로 합산
  const catIds = new Set(
    budgets.filter((b) => b.categoryId !== null).map((b) => b.categoryId!),
  );
  let sum = 0;
  for (const id of catIds) sum += budgetForCategory(budgets, ym, id);
  return sum;
}
```

- 대시보드(`totalBudget`)·분석 탭(`budgetForCategory`)의 예산 표시는 이 규칙을 자동으로 따른다.
- 어느 카테고리도 예산이 없으면 `budgetForCategory`는 0 반환(현행과 동일).

## 5. UI (예산·목표 탭 `Plans.tsx`)

예산 영역을 두 섹션으로 나눈다.

```
[기본 예산]  · 매달 적용
 🍚 식비        30만   [수정]
 🧺 생활용품     20만   [수정]
 🍼 육아         15만   [수정]
 합계 기준 예산 65만

[2026-06 이번 달 조정]          ← 보는 달에 오버라이드가 있을 때만 노출
 🍚 식비   40만  (기본 30만)  [기본으로]
```

### 기본 예산 섹션
- 카테고리별 기본 예산 목록(진행률 바 포함, 사용액은 보는 달 기준)
- 각 행 **"수정"** → 예산 편집 시트
- **"+ 설정"**으로 새 카테고리 예산 추가

### 편집 시트 (`forms.tsx` 예산 폼 확장)
- 카테고리 선택 + 금액 입력
- **적용 범위 선택** (라디오/토글):
  - `기본 예산` → `yearMonth = null`로 저장(매달 적용)
  - `이번 달만(2026-06)` → `yearMonth = ym`로 저장(오버라이드)
- 기존 값 수정 시 현재가 기본인지 오버라이드인지 초기 선택에 반영

### 이번 달 조정 섹션
- 보고 있는 달(`ym`)에 오버라이드가 있는 카테고리만 표시
- 각 행: `카테고리 · 오버라이드 금액 (기본 N만)` + **"기본으로"** 버튼
- "기본으로" = 그 달 오버라이드 레코드 삭제 → 기본값으로 복귀

### 진행률
- 사용/예산 비율은 **실제 적용값**(`budgetForCategory(budgets, ym, id)`) 기준

## 6. 삭제 규칙

- **기본 예산 삭제**(기본 섹션의 삭제): 그 카테고리 기본 예산(`yearMonth=null`) 제거 → 모든 달에서 사라짐. (그 카테고리의 기존 오버라이드가 남아 있으면 그 달만 계속 표시되므로, 기본 삭제 시 함께 정리할지는 구현에서 결정 — 아래 미결)
- **오버라이드 삭제**("기본으로" 버튼): 그 달 오버라이드(`yearMonth=ym`)만 제거 → 기본값으로 복귀. 기본 예산은 유지.

## 7. 범위 (YAGNI)

**포함**: 기본 예산 + 카테고리별 월 오버라이드, 전체 월예산(`categoryId=null`)에도 동일 적용, 조회 헬퍼 수정, 예산·목표 탭 UI(기본/조정 2섹션 + 편집 시트 범위 선택), 마이그레이션.

**제외**: 예산 복사/일괄편집 UI, 예산 히스토리·통계, 기본 예산 변경 시 과거 달 소급, 카테고리별 예산의 요일/주 단위 세분.

## 8. 미결 (구현 시 결정)

- 기본 예산을 삭제할 때, 그 카테고리에 남아 있는 미래·과거 오버라이드를 함께 지울지 vs 남길지. 기본 = **기본 예산 레코드만 삭제하고 오버라이드는 남긴다**(사용자가 명시적으로 만든 값 보존). 이 기본안으로 구현하되 계획서에서 확정.
- 기존 데이터(전부 `"YYYY-MM"` 예산)는 그대로 "그 달 오버라이드"로 해석됨 → 기존 사용자가 기본 예산을 새로 세팅하기 전까지는 현행과 동일하게 동작(회귀 없음).
