# 배추가족 캘린더 개편 — 풀블리드 그리드 · 셀 텍스트 · 자유 카테고리

- 날짜: 2026-07-16
- 대상: 배추가족 PWA `/calendar` 가족 캘린더 (`components/calendar/FamilyCalendar.tsx`)
- 상태: 설계 승인됨 (2026-07-16), 구현 계획 대기

## 배경 / 목표

가족 캘린더가 현재 3가지 불편이 있다.

1. 월 그리드가 `Card`(둥근 박스+패딩) 안에, 다시 `px-4` 여백 안에 있어 화면 대비 작게 보인다.
2. 일정이 **색 점**으로만 표시돼 무슨 일정인지 셀에서 알 수 없다.
3. 카테고리 축이 `담당(추추/배찌/함께)` 고정 enum이라 사용자가 카테고리를 늘리거나 색을 고를 수 없다.

iOS 기본 캘린더처럼 **화면 꽉 찬 그리드 + 셀 안 텍스트 일정 + 자유 카테고리(색 선택)** 로 바꾼다.

## 결정 사항 (승인됨)

- **카테고리 모델**: `담당`을 **자유 카테고리로 대체**. 추추/배찌/함께를 기본 카테고리 3종으로 두고, 그 위에 자유 추가. 일정은 카테고리 1개를 가진다.
- **색 선택 방식**: 앱 톤 팔레트 스와치(10~12색)에서 선택. 컬러 휠 아님(앱 통일감 유지).
- **투두기한**: 카테고리가 아니라 투두 마감 파생 오버레이 → 색 고정, 편집 대상 아님. 범례엔 계속 표기.

## 1. 풀블리드 월 그리드

- 월 그리드를 감싼 `Card` 제거, 좌우 `px-4` 여백을 그리드 구간에서만 제거해 **화면 좌우 끝까지** 채운다. (앱은 `max-w-md` 모바일 프레임 → 폰에서 사실상 뷰포트 전폭)
- 요일 헤더 + 그리드가 전폭. 셀 간 세로 구분선(옅은 `--color-line`)로 iOS 느낌.
- 셀 높이를 키워 날짜 숫자 아래 **텍스트 일정 최대 3줄**이 들어가게 한다. (기존 고정 높이 → 최소 높이 `min-h`)
- `+일정` 버튼, `MonthSwitcher`, 선택 날짜 상세 패널은 지금처럼 여백 있는 카드로 그리드 위/아래 유지.
- 드래그(마우스 즉시 / 터치 롱프레스 0.4초)·탭 상호작용은 셀 `data-iso`·포인터 핸들러 그대로 유지. 셀 안 텍스트 막대는 표시 전용(`pointer-events` 통과).

## 2. 셀 안 텍스트 일정 (색 점 → 텍스트 막대)

- 각 셀: 상단 날짜 숫자 → 하단에 그날 일정들을 **작은 막대**로 세로 나열.
- 막대 1개 = 카테고리 색 배경 + 제목(말줄임, `text-[9~10px]`). 종일이 아니면 앞에 시간 생략(공간상 제목 우선).
- 셀당 **최대 3개** 표시, 초과 시 마지막 줄 `+N`.
- 투두 기한(미완료·dueDate)도 같은 막대 형태로 투두기한(gold) 색, 일정 뒤에 붙여 3칸 예산 공유.
- 막대 텍스트 색: 배경 hex의 밝기(luminance)로 흰색/짙은색 자동 선택(`readableText(hex)` 헬퍼). 밝은 팔레트색(sprout/gold)은 짙은 글씨.
- 오늘 셀 강조·선택 셀 링은 유지하되 날짜 숫자 원형에만 적용(막대 영역과 분리).

## 3. 자유 카테고리 + 색

### 엔티티

```ts
interface EventCategory {
  id: string;          // 'cat-chuchu' 등 고정 or newId()
  name: string;
  color: string;       // hex, 팔레트 스와치 중 하나
  emoji: string | null;// 선택(기본 카테고리는 시드에 이모지 있음)
  sortOrder: number;
  createdAt: string;
```

- `FamilyEvent.assignee: TodoAssignee` → `FamilyEvent.categoryId: string` 로 교체.

### 관리 UI

- **일정 폼**(`EventForm`): 기존 `담당` 토글 → **카테고리 선택줄**(색 스와치 원 + 이름 칩, 가로 스크롤) + 끝에 `+ 관리` 버튼.
- **관리 시트**(`CategoryManager`, 신규): 카테고리 목록(색·이름), 각 행 편집(이름 입력 + 팔레트 스와치 선택)·삭제(2단계 확인). 하단 `+ 새 카테고리`.
- 카테고리는 **하드 삭제**(budget 카테고리 패턴, 휴지통 없음).

### 삭제 규칙

- 카테고리를 쓰는 일정이 N개면 삭제 확인에 "이 카테고리 일정 N개는 '○○'(다음 카테고리)로 옮겨집니다" 경고 → 확정 시 해당 일정 `categoryId` 재배정 후 삭제(고아 방지).
- 카테고리가 1개뿐이면 삭제 불가(최소 1개 보장).

## ⚠️ 배포 순서 함정 (조용한 유실)

`sbUpsert`(lib/repo.ts)는 error를 검사하지 않는다. 스키마와 코드가 어긋나면 크래시가 아니라 **조용한 유실**이 된다.

- **DDL 먼저, 배포 나중.** `event_categories` 테이블 + `family_events.category_id`가 없는 상태로 새 코드가 배포되면 저장이 조용히 사라진다.
- DDL 적용은 재헌님이 **명시적으로 승인**해야 진행(auto mode classifier가 프로덕션 DDL 차단). 개인 토큰 `~/.claude.json` `SUPABASE_ACCESS_TOKEN`(sbp_) → Management API `/database/query`, node https, `dangerouslyDisableSandbox: true`.
- "푸시"는 DDL 승인이 아니다.
