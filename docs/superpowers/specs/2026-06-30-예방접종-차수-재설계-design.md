# 예방접종 차수(몇차) 재설계 + 배추 실제 내역 임포트

- 작성일: 2026-06-30
- 대상: 배추가족 PWA — 배추 건강 탭 / 예방접종
- 선행: 2026-06-30 예방접종 체크리스트(0013, 매년 부스터 단순형) — 이를 차수 기반으로 확장

## 배경 / 동기

배추의 실제 동물병원 접종 내역은 백신마다 1차~N차로 나뉜다(종합백신처럼 차수가 비연속 5·6·7·8차인 경우도 있음). 기존 모델(`history: string[]` 단순 날짜배열 + 매년 부스터 자동)로는 차수를 표현·관리할 수 없다. 차수별 접종일을 보고 입력할 수 있는 구조로 바꾸고, 사진에서 읽은 배추 실제 내역을 DB에 직접 적재한다.

## 결정 사항 (확정)

| 항목 | 결정 |
| --- | --- |
| 차수 모델 | `doses: { n: number; date: string }[]` (차수 + 접종일). 비연속 차수 허용 |
| 최근 접종일 | doses에서 파생(별도 컬럼 없음). 단일 진실원천 |
| 다음 예정일 | `nextDue` 별도 저장(사진의 접종예정일, 자동 파생 아님) |
| 카드 ✓ | 다음 차수(max n+1)를 오늘 날짜로 추가 + nextDue = 오늘+주기 |
| 차수 UI | 카드 요약 + 상세시트(차수 목록 관리) |
| 프리셋 | 배추 실제 7종 이름으로 교체 |
| 데이터 | 기존 테스트 5행 삭제 → 실제 7종 임포트 |

## 데이터 모델

### TS (`lib/types.ts`)

```ts
export interface VaccineDose {
  n: number;      // 차수 (1, 2, ... 비연속 가능)
  date: string;   // 접종일 (YYYY-MM-DD)
}

export interface BaechooVaccine {
  id: string;
  name: string;
  nextDue: string | null;     // 다음 예정일 (YYYY-MM-DD, 없으면 null)
  intervalMonths: number;     // 추가접종 주기(개월, 기본 12)
  doses: VaccineDose[];       // 차수별 접종 기록
  memo: string | null;
  createdAt: string;          // YYYY-MM-DD
}
```

`lastDone`·`history` 필드 제거. `DataSnapshot.baechooVaccines`는 유지.

## 파생 헬퍼 (`lib/format.ts` 또는 컴포넌트 내)

- `latestDose(doses)`: date가 가장 늦은 dose. 없으면 null. (최근 접종 표시·정렬용)
- `nextDoseNumber(doses)`: `max(n) + 1` (없으면 1). 카드 ✓ / 차수 추가 기본값.

## UI

### 카드 (`VaccineList`)

```
[✓]  켄넬코프백신                       [D-362]
     최근 5차 · 2026-06-27
```

- 최근 접종 = `latestDose`. 표시: `최근 {n}차 · {date}`. doses 비었으면 "접종 기록 없음".
- D-day Pill: nextDue 기준. 톤 = 지남 coral / ≤30 gold / 그외 stone. nextDue 없으면 "예정 없음" stone. doses 비고 nextDue 없으면 "미접종" stone.
- 정렬: nextDue asc(null 뒤로) — 기존과 동일.
- ✓ = `complete(v)`: nextDoseNumber로 오늘 dose 추가 + nextDue = addMonths(오늘, interval).
- 이름/요약 탭 → 상세시트.

### 상세시트 (`VaccineDetailSheet`, forms.tsx) — 신규 백신/편집 공용

한 시트에서 백신 전체 관리:
1. **이름** (text)
2. **차수 목록**: 각 행 = 차수(number, text+inputMode) + 접종일(date) + 행 삭제. 차수 asc 정렬 표시. "+ 차수 추가"(기본 n = nextDoseNumber, date = 오늘).
3. **다음 예정일** (date, 선택)
4. **추가접종 주기 (개월)** (text+inputMode numeric, 기본 12)
5. **메모** (선택)
6. 저장 / (편집 모드) 백신 2단계 삭제

저장 시 빈 날짜의 차수 행은 제외. 차수는 입력 순서 무관, 저장 시 n asc 정렬.

기존 `VaccineForm`은 이 시트로 대체(이름 변경 또는 확장).

## 데이터 임포트 (1회성, Management API node https)

1. 기존 테스트 5행 영구 삭제(`delete from baechoo_vaccines`).
2. 배추 실제 7종 insert. id=gen_random_uuid 또는 앱 newId 형식(text). doses는 jsonb.

| 백신 | doses (n:date) | next_due |
| --- | --- | --- |
| 켄넬코프백신 | 1:2023-06-15, 2:2023-06-29, 3:2024-06-20, 4:2025-06-20, 5:2026-06-27 | 2027-06-27 |
| 코로나장염백신 | 1:2023-05-30, 2:2023-06-15, 3:2024-06-20, 4:2025-06-20, 5:2026-06-27 | 2027-06-27 |
| 종합백신 | 5:2023-05-30, 6:2024-06-20, 7:2025-06-20, 8:2026-06-27 | 2027-06-27 |
| 광견병백신 | 1:2023-05-15, 2:2024-05-17, 3:2025-05-02 | 2026-05-02 |
| 심장사상충 | 1:2023-05-24, 2:2023-08-31 | null |
| 외부기생충 | 1:2023-08-11 | 2023-11-11 |
| 신종인플루엔자백신 | 1:2023-06-29, 2:2023-07-24 | 2024-07-23 |

interval_months=12, memo=null, created_at=now.
한글 INSERT는 node https + UTF-8(`Buffer.from`)로 안전 전송(PowerShell mojibake 회피).

## 비범위 (YAGNI)

- 차수별 병원/메모 — 차수는 {n,date}만. (병원명은 백신 단위 memo로)
- 접종예정일 자동 산출 규칙(퍼피 시리즈 간격) — 수동 입력 + 부스터 +interval만.
- 건강 기록 연동 — 여전히 안 함.

## 호환성 메모

기존 0013 데이터는 테스트 5행뿐 → 임포트에서 전부 삭제하므로 history→doses 데이터 마이그레이션 불필요. 코드 경로(localStorage `lsRead`의 구버전 `baechooVaccines`)도 doses 없으면 빈 배열로 안전 처리.
