# 배추가족 가족 P&L — 회계모델 규칙서

> 가족 재무를 경영자 관점 손익계산서로 보는 규칙 정본
> 이 문서가 2단계 PWA 손익탭의 로직 명세가 된다
> 최종 확정: 2026-07-03

---

## 1. 손익 구조 (하이브리드)

기업 손익계산서의 **용어**를 쓰되, 원가/판관비의 **경계는 고정비/변동비**로 정의한다.

| 순서 | 항목 | 뜻 | 예시 |
|---|---|---|---|
| | **매출 (총수입)** | 그 달 들어온 모든 돈 | 급여, 기타수입 |
| − | **매출원가 (고정비)** | 안 줄어드는 필수·부채상환 | 주거, 공과금, 보험, 구독, 할부, 청약 |
| − | **선저축·적립투자** | 미래 몫을 원가처럼 먼저 뗀다 | 적금, 목표저축(Goal) 적립 |
| = | **가처분이익** | 변동비에 쓸 수 있는 재원 | |
| − | **판관비 (변동비)** | 줄일 수 있는 생활비 | 식비, 외식, 쇼핑, 여가, 병원, 육아, 배추 |
| = | **운영이익 (잉여)** | 저축까지 하고도 남은 진짜 여윳돈 | |

### 핵심 철학
- **선저축**: 저축은 "남으면 하는 것"이 아니라 필수 비용. 고정비 다음에 먼저 확보한다.
- **운영이익 = 계획 저축까지 마친 뒤의 잉여**. 재무 건전성을 보수적으로 본다.

---

## 2. 지표

| 지표 | 공식 | 의미 |
|---|---|---|
| 총이익률 | 가처분이익 ÷ 매출 | 고정비·저축 뗀 뒤 남는 비율 |
| **운영이익률** | 운영이익 ÷ 매출 | 최종 여윳돈 비율 (핵심 KPI) |
| **BEP (손익분기)** | 운영이익 = 0 | 고정비 + 계획저축 + 변동비를 수입으로 딱 감당하는 상태 |

- **운영이익 > 0** → 계획한 저축을 다 하고도 흑자 (BEP 달성 + 여유)
- **운영이익 = 0** → BEP. 저축은 지켰지만 여윳돈은 없음
- **운영이익 < 0** → 적자. 계획 저축을 지키지 못했거나 변동비 초과

---

## 3. 카테고리 → 손익항목 매핑

현재 가계부 데이터로 자동 분류한다. 판정 우선순위: **① 고정지출 등록 여부 → ② 카테고리 → ③ 습관태그**.

| 손익 항목 | 매핑 대상 | 판정 근거 |
|---|---|---|
| 매출 (수입) | 급여, 기타수입 | `Category.type === "income"` |
| 매출원가 (고정비) | 고정지출(`RecurringExpense`: 고정·구독·할부), 청약, 할부 | 고정지출 등록 / 부채상환·주택청약 |
| 선저축·적립투자 | 적금, 목표저축(Goal) 적립분 | 저축성 항목 |
| 판관비 (변동비) | 식비·생활용품·육아·병원·취미·배추·기타 | 나머지 지출. 습관태그(배달·커피·외식·쇼핑·택시·구독)로 "줄일 수 있는 변동비" 세분 |
| **손익 제외** | 카드값 | 결제수단 정산 → 각 카테고리 지출과 이중계상되므로 제외 |

### 매핑 원칙
- 자동 판정을 기본값으로 하되, 카테고리마다 손익분류를 수동으로 바꿀 수 있게 한다(2단계 PWA).
- **카드값은 손익에서 뺀다** — 실제 지출은 이미 각 카테고리에 기록돼 있고 카드값은 그 합산 정산이므로, 넣으면 두 번 센다.

### 확정된 운용 규칙 (2026-07-03)
- **카드값 = 실제 사용액만 기록, 명세서 총액은 넣지 않음**. 각 지출을 카테고리별로 실제 금액으로 직접 다는 방식이라 이중계상이 원천 차단돼 있다. 따라서 손익 집계에서 카드값(명세서 정산)은 그대로 무시·제외한다.
- **청약 = 고정비(매출원가)** 확정. 선저축으로 옮기지 않는다.

---

## 4. 자산 스냅샷 대조

손익(기간 흐름)과 잔액(시점 자산)은 다른 표다. P&L을 중심으로 하되 월말 자산을 경량 연결한다.

- 매월 말 **총자산**(현금 + 계좌 + 투자 잔액 합계)을 수동 입력
- **운영이익(잉여) ≈ 월말 총자산 − 월초 총자산** 을 대조
- 차이가 크면 거래 누락·미기록 신호로 점검

```
운영이익(잉여)       +500,000
        ↓ 대조
월초 총자산       50,000,000
월말 총자산       50,480,000
실제 증가           +480,000
차이 20,000 → 누락 점검
```

> 순자산(자산−부채) 추이, 부채·자산 상세 관리는 범위 밖(2단계 이후 별도 확장).

---

## 5. 시간축

- **월 뷰**: 해당 월의 P&L, BEP 달성 여부, 운영이익률
- **연 뷰**: 12개월 누적 + 월별 추이

## 6. 예산 계획 (계획 탭)

계획은 실적과 **같은 4버킷**(매출·고정비·선저축·변동비)으로 집계한다. 항목마다 손익분류를 직접 지정하므로, 카테고리 자동규칙에 의존하지 않는다. 그래서 계획과 실적을 나란히 놓고 뺄 수 있다.

- **구획(group)** = 화면상의 자리 — 수입 / 월 지출 / 저축·상환
- **손익분류(pnlClass)** = 손익 집계 버킷 — 매출 / 고정비 / 선저축 / 변동비
- **둘은 독립이다.** 학자금 상환은 구획이 `저축·상환`이지만 손익분류는 `고정비`다.
- **여유 시 집행(⊖)** = 최우선이 아닌 항목. 부모급여·아동수당이 들어오면 집행한다. 손익 집계에는 정상 포함되고, 화면에서만 별도 섹션으로 묶인다.

계획값은 **실제 청구액이 아니라 넉넉하게 올려 잡은 값**이다. 계획 월세는 600,000이지만 실제 정기지출은 539,840이다. 그래서 계획(`plan_items`)과 정기지출(`recurring_expenses`)을 한 테이블에 섞지 않는다.

### 집행 기간

항목은 **시작월 ≤ 기준월 ≤ 종료월**일 때 그 달의 계획에 잡힌다(null = 무한대).

| 종류 | 시작월 | 종료월 | 예 |
|---|---|---|---|
| 매달 | null | null | 월세·식비 |
| 언젠가 끝남 | null | 있음 | 자동차 할부 (2027-03까지) |
| **1회성** | **= 종료월** | **= 시작월** | 어머니 산후조리비 (2026-08에만) |

1회성은 별도 플래그가 아니라 **시작월 = 종료월**이라는 파생 조건이다. 그래서 판정 규칙이 한 줄로 유지된다.

### 다가오는 1회성 지출

미래에 꽂아둔 1회성은 「다가오는 1회성 지출」 카드에 보이지만 **그 달로 넘어가야 잔액에 반영된다.** 이번 달 잔액에는 안 잡힌다.

지난 1회성은 자동으로 지우지 않는다 — 과거 계획 vs 실적 기록이 보존된다.

### 기준선

매달 반복분만의 계획 운영이익 = **25,794원** (재헌님 손글씨 예산 노트의 "잔액"과 원 단위 일치).
1회성이 있는 달은 그만큼 떨어진다.
